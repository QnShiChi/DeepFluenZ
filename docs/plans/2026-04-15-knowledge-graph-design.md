# Subject Knowledge Graph (Hybrid Mode) Design

## Overview
Transform the linear curriculum of DeepTutor into an intelligent, adaptive subject map. Instead of a rigid course structure, the learning path becomes an interactive "Skill Tree" where the core nodes are generated by AI from standard syllabus documents, and "side quests" are dynamically generated to plug students' knowledge gaps.

## Data Architecture & Representation 

We will use JSON/Relational hybrid structures to avoid the overhead of a dedicated GraphDB.

1. **CourseGraphTemplate (The Base Map):**
   - The master copy representing a subject's core curriculum.
   - Contains a list of `KnowledgeNode`.
   - Each `KnowledgeNode` has properties: `node_id`, `type` (e.g., MAIN), `title`, `description`, `dependencies` (list of node IDs required to unlock).

2. **StudentGraphState (The Persona State):**
   - Tracks individual student progress on the graph.
   - Stores `current_node_id`, `mastered_nodes`.
   - **Dynamic Spawning:** A `dynamic_nodes` JSON field storing personalized "Side Quests". These are sub-nodes injected solely for this student when they fail a main node concept.

## AI Generation Workflow 

1. **Main Quests Generation:**
   - Instructors upload a syllabus PDF.
   - A `GraphExtractionAgent` uses LLM structure queries to parse the text and spit out the list of `KnowledgeNode` definitions and dependencies.
   
2. **Dynamic Spawning Engine (Side Quests):**
   - Hooks into `stream_bus.py`. When the `exam_completed` event fires, it checks the score.
   - If score < 50%, `deep_reason` analyzes the student's exact mistakes.
   - The engine generates a new `KnowledgeNode` with `type=SIDE_QUEST` (e.g., "Refresher on Pointer Management") and appends it to their `StudentGraphState.dynamic_nodes`.
   - The student must pass this side quest before advancing to the next Main node.
